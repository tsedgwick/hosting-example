libmongo

     mkdir -p etc/dylib

Getting Started

Backend Setup

Run the following to get the source code set up for the backend:

mkdir stitch && cd stitch && export GOPATH=`pwd`
mkdir -p $GOPATH/src/github.com/10gen
cd $GOPATH/src/github.com/10gen
git clone git@github.com:10gen/stitch.git
cd stitch

Transpiler

You will also need to set up the transpiler to start the server. You can build the transpiler by running the following.

cd etc/transpiler
yarn && yarn build

Then you must place the resulting binary found in etc/transpiler/bin/transpiler into a directory that is on your system $PATH.

Code Generation

For certain models code, we use code generation to automatically produce boilerplate code so that it does not need to be written by hand. First build the code generator binary with go build -o $GOPATH/bin/model github.com/10gen/stitch/models/codegen. This will build a binary called model into $GOPATH/bin; copy this binary to a directory that is included in your $PATH. To use it for generating a model, start with an interface type, such as:

type Foo interface {
  FieldAAA() int
  FieldBBB() string
  Builder() *FooBuilder
}

Add an annotation after the type definition: //go:generate model basic -type=Foo Then run go generate from within the same directory as the source file containing the interface type. This will write a basic implementation of the model and a builder to foo_gen.go. You should commit this file to source control but do not edit it manually; re-run go generate as needed to keep it up to date with model changes.

You will also need to implement a prebuild function; this is called internally by the builder when creating new instances of the model, allowing you to throw errors, pre-populate data, etc. It must have a signature of the following form: func (*FooBuilder) prebuild(built *basicFoo) error

Starting the Server

Before starting the stitch server, you should have a replica set (on version 3.4 or greater) running at localhost:27017. To do this, run:

mongod --replSet local --fork --logappend --logpath ~/db/logsmongod.log --dbpath ~/db/data/

The first time you start it up, connect to the mongod instance with the shell and run rs.initiate().

Then from $GOPATH/src/github.com/10gen/stitch, run: go run cmd/server/main.go --configFile ./etc/configs/local_config.json

This will run Stitch on 8080 with the following settings:

  • Metadata stored on MongoDB at mongodb://localhost:27017

To create a seed user to log in, run: go run cmd/auth/user.go addUser -domainID 000000000000000000000000 -mongoURI mongodb://localhost:27017 -salt 'DQOWene1723baqD!_@#' -id "unique_user@domain.com" -password "password"

To have the UI load, see Frontend instructions and yarn run build. You may need to restart the server for the changes to take effect.

With Cloud Manager

Hey wait! If you don't want to set up any of the below and just want to work against cloud-dev, start Stitch with ./etc/configs/local_cloud_dev_config.json. Saved you a few minutes.

You will need java, ant, and node installed; see https://wiki.corp.mongodb.com/display/MMS/Cloud+Developer+Setup#CloudDeveloperSetup-Java

First set up Cloud Manager:

cd ~
mkdir mms
git clone git@github.com:10gen/mms.git
cd mms/server

Then add your IP to the temporary API key whitelist, by creating the file conf/conf-local-secure.properties with contents:

mms.publicApi.globalWhitelist=0:0:0:0:0:0:0:1

Then run the server:

ant mms.server.local

Cloud Manager is now running at port 8080.

Create an API Key via the locally running cloud UI at at /v2#/account/publicApi and save it somewhere.

Then, create a second JSON config file in a location of your choosing containing the following data (be sure to provide your own adminUsername and adminAPIKey that you just created):

my_cloud_config.json

{
  "admin": {
    "authProviders": [
      {
        "name": "mongodb-cloud",
        "type": "mongodb-cloud",
        "config": {
          "publicAPIBaseUrl": "http://localhost:8080/api/public/v1.0",
          "atlasAPIBaseUrl": "http://localhost:8080/api/atlas/v1.0",
          "privateAPIBaseUrl": "http://localhost:8080/api/private",
          "adminUsername": "<ENTER USERNAME HERE>",
          "adminAPIKey": "<ENTER API KEY HERE>"
        },
        "metadataFields": [{ "name": "name", "required": false}]
      }
    ]
  }
}

From $GOPATH/src/github.com/10gen/stitch, run: go run cmd/server/main.go --configFile ./etc/configs/local_cloud_config.json --configFile path/to/my_cloud_config.json

Stitch is now running at port 9090.

  1. Go to Cloud Manager and log in (credentials given to you by your lead).
    1. If you're in the Admin UI of Cloud, click your name in the top right and go to profile.
    2. Click organizations and click New Organization and make an Atlas organization.
    3. Make a new project.
  2. Create a stitch app.

Frontend

With the backend code already checked out, from its root directory ($GOPATH/src/github.com/10gen/stitch) run:

mkdir -p static && cd static
git clone git@github.com:10gen/stitch-ui.git app
cd app
yarn install

From here, you can use:

  • yarn run build which generates a webpack bundle that will be served by the back-end at http://localhost:8080.
  • yarn start to launch a webpack-dev server which serves the front-end code on http://localhost:8001. Use this for doing front-end development.

General

Testing

Backend Tests

Backend tests can be run using:

go test -tags debug -parallel 1 -p 1 $(go list ./... | grep -v '/\(vendor\|cmd\|etc\)/')

The debug build tag enables dynamic type assertion in some places where we cannot rely on the compiler for type enforcement. It should be used only when testing and not in production.

Tests that need some general mongo functionality should import github.com/10gen/stitch/utils/test. This will also provide a STITCH_MONGODB_URI environment variable for setting the database to use during testing (mongodb://localhost:26000 is the default).

Example:

STITCH_MONGODB_URI="mongodb://localhost:27017" go test -v -tags debug ./api/admin -run MongoDB

To test a specific package, for example the "client" package:

go test -v -tags debug github.com/10gen/stitch/api/client

To run specific test functions, you can provide a regex:

go test -v -tags debug github.com/10gen/stitch/api/client -run 'TestDeleteService$'

Front-end tests

See stitch-ui.

Debugging

To debug the backend code, you can use delve. If using VSCode, we have ready-made configurations located in /etc/contrib/vscode configs that should work out of the box. Copy those to the .vscode folder in the root of the workspace. Then make sure dlv is installed (CMD+Shift+P, then Go: Install/Update Tools, select dlv and click Ok). The currently supported debug configurations are:

  • Debug Server launches the server and attaches a debugger - it's equivalent to go run cmd/server/main.go --configFile ./etc/configs/local_config.json.
  • Debug Package Tests will launch the tests for the package of the currently open file. For example, if you're editing facebook_test.go, it will execute a command that is identical to go test github.com/10gen/stitch/authprovider/providers/oauth2/facebook -v (because facebook_test.go is located in the authprovider/providers/oauth2/facebook folder).

Dependencies

Backend

Backend dependencies are vendored in the vendor/ directory, and are managed with the dep tool, which maintains the versioning information in the Gopkg.toml and the Gopkg.lock files. To install dep:

macOS:

brew install dep

or

Other Operating Systems:

go get -u github.com/golang/dep/cmd/dep

See https://github.com/golang/dep for more usage instructions.

Contributing

For an overview of the development workflow, see etc/contrib/workflow.

We use golangci-lint to lint our code on Evergreen. If you'd like to enforce this locally before committing, copy the etc/contrib/pre-commit git hook to your .git/hooks directory.

To get golangci-lint, first make sure that $GOPATH/bin is in your path. Then run the following:

go get -u github.com/golangci/golangci-lint/cmd/golangci-lint

To run it manually from the stitch source directory:

(export PKGS=`go list ./... | grep -v "/\(vendor\|etc\/transpiler\/node_modules\|etc\/cpp\)/"`; cd $GOPATH/src && echo $PKGS | xargs $GOPATH/bin/golangci-lint run)

For more information on golangci-lint and configuration details - https://github.com/golangci/golangci-lint
